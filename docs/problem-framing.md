# Problem Framing: Product Analytics Dashboard dengan WebGIS

## 1. Latar Belakang

Perusahaan retail memiliki dataset produk yang tersebar di berbagai lokasi di Pulau Jawa. Data ini mencakup informasi produk, penjualan, biaya, dan lokasi geografis — namun saat ini belum ada alat yang memadai untuk memvisualisasikan dan menganalisis data tersebut secara terpadu.

## 2. Rumusan Masalah

1. **Sebaran produk tidak tervisualisasi** — Tim bisnis tidak dapat melihat di mana saja produk tersedia secara geografis. Tidak ada peta interaktif yang menunjukkan distribusi offline store.

2. **Performa produk tidak terpantau** — Metrik bisnis seperti total revenue, jumlah pesanan, dan biaya produk tidak dapat dimonitor secara cepat. Tidak ada dashboard yang menyajikan gambaran umum kesehatan bisnis.

3. **Produk bermasalah sulit diidentifikasi** — Tidak ada alat untuk mengidentifikasi produk dengan performa rendah, recency tinggi (jarang terjual), atau health score kritis. Keputusan bisnis menjadi lambat dan kurang data-driven.

4. **Data tidak terstandarisasi** — Dataset mengandung ketidakkonsistenan (typo pada nama segmen, field name tidak konsisten) yang menyulitkan analisis manual dan berpotensi menyebabkan kesalahan interpretasi.

## 3. Tujuan

Menyediakan aplikasi yang memungkinkan:

1. **Visualisasi geografis** — Melihat sebaran produk di peta interaktif dengan kemampuan filter berdasarkan kategori, segmen, dan pencarian nama produk.

2. **Monitoring dashboard** — Menyajikan KPI utama (total produk, revenue, pesanan, biaya rata-rata) beserta chart distribusi kategori, segmen, biaya, dan analisis kesehatan produk.

3. **Identifikasi produk bermasalah** — Menampilkan health score dan status produk (Healthy, At Risk, Critical) untuk memudahkan prioritas tindakan bisnis.

4. **Standarisasi data** — Data dinormalisasi pada saat import sehingga analisis konsisten dan akurat.
